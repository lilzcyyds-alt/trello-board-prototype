# 2.1 Inbox Page

## Summary
Provide a simplified Trello-like Inbox view with a list of actionable items/cards.

## Requirements
- Route: `/inbox`
- Layout:
  - Sidebar visible (see 2.2)
  - Main content area shows an Inbox title and list of items
- Interactions:
  - Hover/active states on list items
  - Click item selects it (visual highlight)
  - (Optional) Search input filters mock items

## Acceptance Criteria
- Inbox page renders without errors
- Items are visually aligned with Trello-like spacing and typography
- Selecting/hovering feels responsive and consistent

## Notes
- Inbox items can be backed by the same `Card` model used on Board (mocked)
