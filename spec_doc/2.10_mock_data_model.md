# 2.10 Mock Data Model & State Shape

## Summary
Define a minimal state model to support Inbox + Board + DnD.

## Requirements
- Normalized storage recommended:
  - `Card`: `{ id: string, title: string }`
  - `List`: `{ id: string, title: string, cardIds: string[] }`
  - `BoardData`: `{ boardTitle: string, cards: Record<id, Card>, lists: Record<id, List>, listOrder: string[], inboxIds: string[] }`
- Seed data:
  - 3–5 lists
  - 3–8 cards

## Acceptance Criteria
- State supports:
  - Card reorder
  - Card cross-list move
  - List reorder
  - Modal edits

## Notes
- Keep IDs stable and human-readable (`c1`, `list-inbox`, etc.)
